# EarCoach Evaluation Data

This folder contains the programs and session logs used to validate EarCoach's
hint generation pipeline.

## Structure

```
evaluation/
  programs/          Five buggy Python programs used as evaluation inputs
  logs/              JSONL session logs generated by running EarCoach on them
  README.md          This file
```

## Programs

| File | Bug type | Runtime signal |
|------|----------|----------------|
| `01_off_by_one.py` | `range(len(items) + 1)` causes `IndexError` | `IndexError: list index out of range` |
| `02_type_mismatch.py` | String + int concatenation | `TypeError: can only concatenate str (not 'int') to str` |
| `03_infinite_loop.py` | Missing `n -= 1` in while-loop | Code runner timeout after 5 s |
| `04_wrong_conditional.py` | `n < 10` should be `n > 10` | Silent logic error (no exception) |
| `05_undefined_variable.py` | `running_total` never initialised | `UnboundLocalError: local variable referenced before assignment` |

## Session Logs

`logs/evaluation_run.jsonl` contains 20 hint events (5 programs x 4 repeats)
generated by running `backend/run_evaluation.py` against a live EarCoach backend
(Ollama `llama3.2:1b`, `edge-tts` TTS, macOS audio output).

Each line is a JSON object. Event types:

- `session_start` — backend started, records timestamp and log path
- `hint` — a hint was generated and spoken; includes `file_name`, `trigger`,
  `stuck_score`, `stuck_breakdown`, `latency_ms`, `spoken`, `hint_text`,
  `error_count`, `runtime_error`
- `session_end` — backend shut down, records total events

## Reproducing

1. Start the EarCoach backend:
   ```bash
   cd backend
   source ../venv/bin/activate
   uvicorn main:app --host 127.0.0.1 --port 8000
   ```
2. In a second terminal:
   ```bash
   source ../venv/bin/activate
   python backend/run_evaluation.py
   ```
   Logs are written to `~/earcoach_sessions/`. Copy the resulting file to
   `evaluation/logs/` to reproduce the dataset.

## Requirements

- Python 3.11+, Ollama with `llama3.2:1b` pulled, `edge-tts`, `playsound`
- See `backend/requirements.txt` for the full dependency list
